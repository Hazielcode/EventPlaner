    fun getEvents(): Flow<List<Event>> = callbackFlow {
        val listener = eventsCollection
            .whereEqualTo("userId", userId)
            .addSnapshotListener { snapshot, error ->
                if (error != null) {
                    close(error)
                    return@addSnapshotListener
                }
                trySend(events)
            }
        awaitClose { listener.remove() }
    }

· Cómo evitar depender totalmente de IA


SIEMPRE ENTENDER ANTES DE COPIAR 

Cada vez que la IA me daba código:
1. Lo leía línea por línea
2. Investigaba qué hacía cada parte
3. Probaba modificarlo para ver qué pasaba
4. Solo lo usaba cuando lo entendía completamente

Por ejemplo, cuando vi "callbackFlow", investigué qué era un Flow en Kotlin,
probé ejemplos simples, y recién entonces lo implementé.
Hice experimentos como:
✓ ¿Qué pasa si cambio StateFlow por MutableLiveData?
✓ ¿Puedo usar var en lugar de val en el data class Event?
✓ ¿Qué error sale si no uso suspend en las funciones de Repository?
✓ ¿Cómo se ve si cambio los colores del tema? 
mas que nada por curioso , ya que quiero dedicarme a Kotlin. ANDROID 
un ejemplo de como no depender , es saber arreglar los problemas básicos : 
cuando apareció "Could not resolve firebase", primero intenté:
- Build → Clean Project
- Invalidate Caches
- Cambiar de red WiFi
Y solo después la IA me sugirió bajar la versión del BoM.
